# GitHub Lucas Faustino Dev

## Tipo de referencia

Referencia personalizada.

## Faixa comercial

R$ 300-400.

## Objetivo

Apresentar uma pagina profissional autoral para Lucas Faustino Dev, com portfolio, servicos, prova visual, copy trabalhada e CTA para orcamento.

## Publico

Pequenos negocios, profissionais autonomos e clientes que precisam de uma pagina com visual mais personalizado, moderno e confiavel.

## Nivel de complexidade

Medio.

## Direcao visual

Dark premium, com detalhes dourados, grid sutil, tipografia forte e apresentacao visual mais autoral. A pagina precisa transmitir clareza, confianca, qualidade e valor percebido.

## Estrutura

- Hero
- Portfolio
- Servicos
- Processo
- Valores
- Duvidas
- CTA de contato

## Funcionalidades

Pagina de portfolio/prospeccao com navegacao, secoes comerciais, links para projetos e CTAs de contato.

## Decisoes importantes

- A pagina profissional nao e o unico projeto principal da pasta `Lucas`; ela e uma referencia dentro do trabalho profissional maior.
- Copy deve ser direta, humana e comercial, sem linguagem generica ou corporativa demais.
- Screenshots reais dos projetos devem funcionar como prova visual.
- Deploy, commit e push dependem de aprovacao explicita.
- A versao atual foi publicada em producao a partir da branch `main`, no commit `fd97433`.
- O projeto foi padronizado com `npm`, mantendo `package-lock.json` e removendo `pnpm-lock.yaml`.
- Na Vercel, o framework foi ajustado para Next.js e o Output Directory ficou no padrao automatico do Next.js.
- `.gitignore` foi criado para evitar `node_modules`, `.next`, logs e arquivos temporarios no repositorio.

## Pontos fortes para reaproveitar

- Boa referencia para paginas personalizadas e autorais.
- Sistema visual dark premium com detalhes dourados.
- Estrutura util para portfolio, servicos e conversao via WhatsApp.

## O que evitar repetir

- Headline longa demais no hero.
- Texto generico com cara de IA.
- Efeitos visuais sem funcao ou excesso de brilho.
- Tratar Finia ou outros projetos autorais como produto central do workspace.
